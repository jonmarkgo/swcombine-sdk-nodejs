The `client.datacard` resource provides access to datacards — production licenses that control which entity types a faction can manufacture.

### datacard.create()

Create/assign datacard to production entity

<ParamField path="production_entity_uid" type="string" required>
  The entity UID to assign this datacard to
</ParamField>

<ParamField path="uid" type="string" required>
  Datacard UID
</ParamField>

<ParamField path="unlimited" type="boolean">
  Set to true for unlimited uses (optional, use this OR uses)
</ParamField>

<ParamField path="uses" type="number">
  Number of uses (optional, use this OR unlimited)
</ParamField>

### datacard.delete()

Delete datacard assignment

<ParamField path="production_entity_uid" type="string" required>
  Production entity UID to revoke from (required)
</ParamField>

<ParamField path="uid" type="string" required>
  Datacard UID
</ParamField>

### datacard.get()

Get a specific datacard by UID.
Returns the `Datacard` object directly — not wrapped in a `Page`.
Access properties like `.name` and `.uid` on the result itself.

<ParamField path="uid" type="string" required>
  The uid parameter.
</ParamField>

```typescript
const card = await client.datacard.get({ uid: 'datacard-uid' });
console.log(card.name); // access properties directly, not card.data
```

### datacard.list()

List datacards owned by faction.
Returns a `Page<Datacard>` — access the array of datacards via `.data`.

<ParamField path="factionId" type="string" required>
  The factionId parameter.
</ParamField>

<ParamField path="pageDelay" type="number">
  The pageDelay parameter.
</ParamField>

```typescript
const page = await client.datacard.list({ factionId: '20:123' });
console.log(page.data);   // Datacard[] — items on this page
console.log(page.total);  // total datacards across all pages

for await (const card of page) {
  console.log(card.name); // auto-paginates
}
```
