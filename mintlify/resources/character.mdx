The `client.character` resource gives you access to the core player identity in the Star Wars Combine universe. You can look up characters by UID or handle, read the authenticated user's own profile, exchange in-game messages, transfer credits, and inspect the fine-grained privilege and permission system.

### character.get()

Get character by UID.
Returns the `Character` object directly — not wrapped in a `Page`.

**Requires:** `CHARACTER_READ`

<ParamField path="uid" type="string" required>
  The uid parameter.
</ParamField>

```typescript
const character = await client.character.get({ uid: '1:12345' });
console.log(character.name); // access properties directly, not character.data
```

### character.getByHandle()

Look up a character's UID by their handle (username).
This endpoint is public — no authentication required — and is the primary
way to resolve a handle to the `uid` used by every other character endpoint.

<ParamField path="handle" type="string" required>
  The handle parameter.
</ParamField>

```typescript
const { uid, handle } = await client.character.getByHandle({ handle: 'Dreks Selmur' });
console.log(uid);    // "1:46931"
console.log(handle); // "Dreks Selmur"

// Chain into a full profile fetch (requires CHARACTER_READ)
const character = await client.character.get({ uid });
```

### character.hasPermission()

Check if the API client has a specific permission for this character

<ParamField path="permission" type="string" required>
  Permission scope to check (e.g., 'CHARACTER_READ', 'MESSAGES_SEND')
</ParamField>

<ParamField path="uid" type="string" required>
  Character UID
</ParamField>

```typescript
const canRead = await client.character.hasPermission({ uid: '1:12345', permission: 'CHARACTER_READ' });
if (canRead) {
  const character = await client.character.get({ uid: '1:12345' });
}
```

## Credit Log

### character.creditlog.list()

Get character's credit log (paginated)

**Requires:** `CHARACTER_CREDITS`

<ParamField path="item_count" type="**item\_count**: number" default="50">
  Number of items to retrieve. Default: 50, Max: 1000
</ParamField>

<ParamField path="pageDelay" type="number">
  Milliseconds to wait before fetching each subsequent page. Helps avoid rate limits during auto-pagination.
</ParamField>

<ParamField path="start_id" type="**start\_id**: number">
  Oldest transaction ID threshold (1 = oldest 1000, 0/default = newest 1000)
</ParamField>

<ParamField path="start_index" type="**start\_index**: number" default="1">
  Starting position for pagination (1-based). Default: 1
</ParamField>

<ParamField path="uid" type="string" required>
  The uid parameter.
</ParamField>

```typescript
const creditlog = await client.character.creditlog.list({ uid: '1:12345' });
const moreLogs = await client.character.creditlog.list({ uid: '1:12345', start_index: 51, item_count: 100 });
const oldestLogs = await client.character.creditlog.list({ uid: '1:12345', start_id: 1 });
// Fetch up to 1000 credit log entries at once
const manyLogs = await client.character.creditlog.list({ uid: '1:12345', item_count: 1000 });

// Iterate through all pages automatically
for await (const entry of creditlog) {
  console.log(entry);
}
```

## Messages

### character.messages.create()

Send a message

**Requires:** `MESSAGES_SEND`

<ParamField path="communication" type="string" required>
  Message text content
</ParamField>

<ParamField path="receivers" type="string" required>
  Semicolon-separated list of recipient character handles (max 25). Example: 'handle1;handle2'
</ParamField>

<ParamField path="uid" type="string" required>
  The uid parameter.
</ParamField>

```typescript
// Valid: handles
await client.character.messages.create({
  uid: '1:12345',
  receivers: 'recipient_handle_1;recipient_handle_2',
  communication: 'Hello from the SDK!'
});
```

```typescript
// Invalid: UIDs are rejected by the API and pre-validated by this SDK
await client.character.messages.create({
  uid: '1:12345',
  receivers: '1:12345',
  communication: 'Test'
});
```

### character.messages.get()

Get a specific message

**Requires:** `MESSAGES_READ`

<ParamField path="messageId" type="string" required>
  The messageId parameter.
</ParamField>

<ParamField path="uid" type="string" required>
  The uid parameter.
</ParamField>

```typescript
import { MessageMode } from 'swcombine-sdk';

const messages = await client.character.messages.list({ uid: '1:12345', mode: MessageMode.Received });
const messageId = messages.data[0]?.attributes.uid;

if (messageId) {
  const message = await client.character.messages.get({ uid: '1:12345', messageId });
  console.log(message.communication);
}
```

### character.messages.list()

List messages sent or received by character (paginated)
Note: list responses return `MessageListItem[]` metadata objects and do not
guarantee a `communication` field. Use `get()` for full message content.

**Requires:** `MESSAGES_READ`

<ParamField path="item_count" type="**item\_count**: number">
  The item_count parameter.
</ParamField>

<ParamField path="mode" type="MessageMode">
  Message mode: `MessageMode.Sent` or `MessageMode.Received`. If omitted, returns both sent and received messages.
</ParamField>

<ParamField path="pageDelay" type="number">
  Milliseconds to wait before fetching each subsequent page. Helps avoid rate limits during auto-pagination.
</ParamField>

<ParamField path="start_index" type="**start\_index**: number">
  The start_index parameter.
</ParamField>

<ParamField path="uid" type="string" required>
  The uid parameter.
</ParamField>

```typescript
import { MessageMode } from 'swcombine-sdk';

const allMessages = await client.character.messages.list({ uid: '1:12345' });
const received = await client.character.messages.list({ uid: '1:12345', mode: MessageMode.Received });
const moreMessages = await client.character.messages.list({ uid: '1:12345', mode: MessageMode.Received, start_index: 51, item_count: 50 });

const firstMessageId = received.data[0]?.attributes.uid;
if (firstMessageId) {
  const detail = await client.character.messages.get({ uid: '1:12345', messageId: firstMessageId });
  console.log(detail.communication);
}

// Iterate through all pages automatically
for await (const msg of allMessages) {
  console.log(msg.attributes.uid);
}
```
